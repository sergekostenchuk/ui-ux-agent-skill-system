# NMT Strategy Source Checkpoint

Date: 2026-06-23
External source: https://github.com/zamesin/Next-Move-Theory-Canon-and-Skills

## Decision

Treat `zamesin/Next-Move-Theory-Canon-and-Skills` as an external product-strategy source for the UI/UX Skill System.

It should not replace the existing UI/UX, SEO, Figma, Stitch, progress, validation, or runtime-control skills. Its correct place is before concept, semantic core, value proposition, GTM, and marketing-site production.

## What To Borrow As Ideas

Use the following ideas as design inspiration for original UI/UX Skill System contracts:

1. Product strategy before UI: clarify segment, customer job, value, risk, and validation debt before visual design.
2. User input as hypothesis: facts from the client should be labeled by confidence, evidence, and risk.
3. Helicopter view before intake: show the user what the system understood and what will happen before asking detailed questions.
4. Value proposition as a required handoff before landing-page copy and design variants.
5. GO means GO to validation, not GO to blind implementation.
6. Readability layering: plain-language summary first, then reasoning, then technical detail.
7. Producer-style skills: each product-strategy skill should produce a concrete handoff artifact, not just advice.

## Proposed Integration

Add an optional bridge skill:

`nmt-strategy-bridge`

Suggested route:

`senior-ui-ux-orchestrator -> project intake -> nmt-strategy-bridge -> semantic-core-builder -> concept-prototyper -> ui-ux-pro-max -> three-variant-design -> Stitch/Figma/code`

## Skills From NMT To Map

- `nmt-chat`: product-strategy advisor / senior council input.
- `nmt-diagnose`: existing product/site diagnosis before redesign or rebuild.
- `nmt-market-research`: market and segment input before semantic core.
- `nmt-craft-value-proposition`: value proposition handoff before concept and landing page.
- `nmt-product-requirements`: PRD input for web apps, admin, SaaS, and product workflows.
- `nmt-craft-go-to-market`: landing-page, ad, and GTM logic before marketing implementation.
- `nmt-analyze-interviews`: interview and voice-of-customer analysis before segmentation and copy.

## License Boundary

Do not copy NMT skill text or canon text directly into the UI/UX Skill System package.

Reason: the repository is licensed as CC BY-NC-SA 4.0. The UI/UX Skill System package should keep its own original contracts and may reference NMT as an optional external source or adapter.

Allowed direction:

- reference the repository;
- create an optional integration/bridge;
- use general ideas and independently written implementation;
- record attribution when NMT is used.

Avoid:

- bundling copied NMT text into Apache/npm package;
- adapting their skill text without compatible licensing;
- making NMT mandatory for every UI/UX task.
