# Severity Model

## P0 Critical

Blocks the primary task, prevents access, loses data, or creates serious privacy/security exposure.

## P1 High

Breaks a major workflow, hides key information, blocks keyboard users, causes serious mobile layout failure, or undermines primary conversion.

## P2 Medium

Creates confusion, slows task completion, weakens hierarchy, misses important states, or causes localized responsive issues.

## P3 Low

Polish issue, minor consistency gap, wording friction, or improvement opportunity with limited immediate impact.

## Finding Shape

Each finding needs:

- severity;
- evidence type;
- observed issue;
- user impact;
- fix;
- verification.
