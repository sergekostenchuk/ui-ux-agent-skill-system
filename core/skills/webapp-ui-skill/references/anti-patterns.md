# Web App UI Anti-Patterns

- Marketing hero section as the first screen of an operational tool.
- Decorative cards inside cards.
- Oversized headings inside compact app panels.
- Filters that move data layout on hover.
- Empty states with no next action.
- Error states that discard user input.
- Hidden keyboard focus.
- One-note color palette with no semantic distinction.
- Viewport-based font scaling.
