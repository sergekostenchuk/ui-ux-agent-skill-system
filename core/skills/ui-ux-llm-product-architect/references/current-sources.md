## Benchmarking Method

When looking for current design solutions:

1. Pick the exact product category and workflow, not generic inspiration.
2. Inspect 5-10 live products that users actually use.
3. Capture screenshots at mobile and desktop sizes.
4. Note navigation model, information density, CTA placement, state handling, form patterns, empty/error states, trust signals, and agent/accessibility signals.
5. Extract reusable decisions; do not copy visual identity, copy, layout composition, or proprietary interaction details.
6. Compare against the product's own audience, constraints, and codebase before recommending adoption.

## Avoid As Primary Sources

- Dribbble/Behance shots without real flows.
- Trend posts with no production examples.
- AI-generated mockups with no state coverage.
- Screenshots that ignore accessibility, responsive behavior, or implementation cost.
